‚ö° 4 Critical Risks (Pre-Launch)

R-01: Google Maps API Dependency (Severity: 5/5 ‚Äî EXISTENTIAL)

Risk: Google's "No Competing Service" clause + 30-day data caching limit could terminate MyMaps at any time.

Mitigation:

  • Dual-layer architecture: use Google for display; require user-sourced data confirmation for independent dataset
  • Evaluate Overture Maps Foundation fallback for basemap
  • Legal review of "competing mapping service" definition pre-launch

R-02: Location Data Privacy (Severity: 4/5 ‚Äî HIGH)

Risk: GDPR/CCPA/FTC enforcement wave targets location data collection.

Mitigation:

  • Location minimization: never log precise GPS to servers
  • Granular consent architecture pre-launch
  • Data processing agreements (DPA) for restaurant-side B2B ideas

R-03: Cold-Start Death Spiral (Severity: 4/5 ‚Äî HIGH)

Risk: Two-sided market: restaurants won't engage until diners exist; diners won't use until restaurants exist.

Mitigation:

  • Curated launch (one city, 200+ manually curated pins, 3-5 influencer pre-seeds)
  • Invert value prop: make app useful for solo user (personal tracker) before social layer

R-04: Incumbent Response (Severity: 3-4/5 ‚Äî MEDIUM-HIGH)

Risk: Google Maps, Yelp, OpenTable can copy features in 6-12 months.

Mitigation:

  • Move fast to market (MVP in 6-8 weeks)
  • Build community + trust moat (creator endorsements, local expertise)
  • Focus on "trust network" defensibility, not just features
